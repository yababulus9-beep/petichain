# Digital Petition Platform with Blockchain Verification

## Overview

This pull request introduces **Petichain**, a comprehensive on-chain petition platform built on Stacks blockchain using Clarity smart contracts. The system enables transparent, tamper-proof petition creation and digital signature verification with full auditability.

## Features Implemented

### ✅ Core Functionality
- **Petition Creation**: Complete petition management with metadata, deadlines, and categorization
- **Digital Signature Verification**: Cryptographic validation of petition signatures
- **Anti-Duplicate Protection**: Prevents multiple signatures from the same principal
- **Real-time Progress Tracking**: Live signature counts and completion percentages
- **Administrative Controls**: Emergency functions for fraud prevention

### ✅ Smart Contract Architecture
- **petition-manager.clar** (323 lines): Core petition lifecycle management
- **signature-validator.clar** (360 lines): Signature verification and storage

### ✅ Security Features
- Comprehensive input validation and error handling
- Deadline enforcement preventing expired signatures
- Authorization controls for administrative functions
- Signature replay protection with petition-specific context
- Immutable petition data once created

## Technical Implementation

### Petition Manager Contract
- **Data Storage**: Efficient mapping of petition IDs to complete petition records

## Data Structures

### Petition Object
```clarity
{
  id: uint,
  title: (string-ascii 200),
  description: (string-utf8 1000), 
  creator: principal,
  target-signatures: uint,
  current-signatures: uint,
  deadline: uint,
  created-at: uint,
  status: (string-ascii 20)
}
```

### Signature Record
```clarity
{
  signature: (buff 65),
  signed-at: uint,
  verified: bool,
  signature-hash: (buff 32),
  block-height: uint
}
```

## Key Functions

### Public Functions
- `create-petition`: Create new petition with validation
- `sign-petition`: Submit verified signature to petition
- `verify-existing-signature`: Re-validate stored signatures
- `cancel-petition`: Admin/creator petition cancellation
- `increment-signature-count`: Update petition progress

### Read-Only Functions
- `get-petition`: Retrieve complete petition data
- `get-petition-progress`: Calculate completion percentage
- `has-signed`: Check if principal has signed
- `get-petition-signature-stats`: Comprehensive signature analytics
